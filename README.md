# 対話コーパス・評価データセット

日本人同士の会話を模した対話コーパスと、長期記憶評価用データセットです。

## ファイル構成

```
conversation_corpus/
├── conversation_corpus.json  # 対話コーパス
└── evaluation_data.json      # 評価用データ
```

## 対話コーパス

### 概要

| 項目 | 値 |
|------|-----|
| ファイル | `conversation_corpus/conversation_corpus.json` |
| 対話数 | 5,000 |
| 発話数 | 10,000 |
| 重複対話 | 0件 |
| 重複発話 | 0件 |

### データ構造

```json
[
  {
    "user1": "発話1（話しかける側）",
    "user2": "発話2（応答する側）"
  },
  ...
]
```

### 特徴

- 日本人同士のカジュアルな日常会話を模した対話
- 多様な話題（温泉、映画、旅行、料理、仕事、健康、趣味など）
- 全対話・全発話がユニーク（重複なし）

### サンプル

```json
{
  "user1": "温泉行きたいな、疲れ取りたい",
  "user2": "箱根とかどう？近いし気軽に行ける"
}
```

```json
{
  "user1": "最近本読んでる？",
  "user2": "積読がマジでことになってる、消化できないかも"
}
```

## 評価用データ

### 概要

| 項目 | 値 |
|------|-----|
| ファイル | `conversation_corpus/evaluation_data.json` |
| 質問数 | 100 |
| 質問文字数 | 10〜71文字（平均45.2文字） |
| 重複質問 | 0件 |

### 用途

質問文を入力として、対応する対話（answer）を正しく検索・取得できるかを評価します。

### データ構造

```json
[
  {
    "question": "質問文（カジュアルな日本語）",
    "answer": "user1の発話 / user2の発話"
  },
  ...
]
```

### 特徴

- 質問文はカジュアルな日本人らしい言葉遣い
- 文字数は10〜100文字の範囲でランダム
- 全answerは対話コーパス内の対話と完全一致

### サンプル

```json
{
  "question": "温泉の話覚えてる？",
  "answer": "温泉行きたいな、疲れ取りたい / 箱根とかどう？近いし気軽に行ける"
}
```

```json
{
  "question": "そういえばさ、カフェについて前に話してたよね。あれどんな内容だった？",
  "answer": "カフェで読書するの好き、雰囲気いいね / コーヒー飲みながらって最高の贅沢"
}
```
